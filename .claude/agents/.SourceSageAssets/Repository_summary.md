## タスク実行の手順

### ドキュメントを作成する場合（基本フロー）

1. 作業内容を分析する
2. **まず日本語でドキュメントを作成する** (`{FILENAME}_JA.md`)
3. **次に英語に翻訳する** (`{FILENAME}.md`)
4. 両方のファイルに言語切り替えバッジを追加する
5. フォーマットとリンクを確認する

### プルリクエストからREADMEを作成する場合

1. PRの差分を取得する
2. 変更されたファイルを分析する
3. プロジェクトの概要を把握する
4. **README_JA.md（日本語）を作成する**
   - プロジェクト名
   - 概要
   - 機能
   - インストール方法
   - 使用方法
   - ライセンス
   - 言語切り替えバッジ
5. **README.md（英語）に翻訳する**
   - 同じ構成で英語版を作成
   - 言語切り替えバッジを含める
6. 両方のファイルをコミットする

### 既存ドキュメントを翻訳する場合

1. 翻訳対象のドキュメントを取得する
2. 内容を理解する
3. もう一方の言語に翻訳する
4. フォーマットを確認する
5. 両方のファイルを更新する

### 翻訳スタイル

**すべてのドキュメントは二言語で作成します:**

- 日本語: `{FILENAME}_JA.md`
- 英語: `{FILENAME}.md`

**翻訳のガイドライン:**
- **元の意味を正確に伝える**
- **自然な表現にする**（日本語も英語も）
- **専門用語は適切に訳す**
- **フォーマットを維持する**
- **コード例は原文のまま保持する**

## タスク実行の手順

### ドキュメントを作成する場合（基本フロー）

1. 作業内容を分析する
2. **まず日本語でドキュメントを作成する** (`{FILENAME}_JA.md`)
3. **次に英語に翻訳する** (`{FILENAME}.md`)
4. 両方のファイルに言語切り替えバッジを追加する
5. フォーマットとリンクを確認する

### プルリクエストからREADMEを作成する場合

1. PRの差分を取得する
2. 変更されたファイルを分析する
3. プロジェクトの概要を把握する
4. **README_JA.md（日本語）を作成する**
   - プロジェクト名
   - 概要
   - 機能
   - インストール方法
   - 使用方法
   - ライセンス
   - 言語切り替えバッジ
5. **README.md（英語）に翻訳する**
   - 同じ構成で英語版を作成
   - 言語切り替えバッジを含める
6. 両方のファイルをコミットする

### 既存ドキュメントを翻訳する場合

1. 翻訳対象のドキュメントを取得する
2. 内容を理解する
3. もう一方の言語に翻訳する
4. フォーマットを確認する
5. 両方のファイルを更新する
