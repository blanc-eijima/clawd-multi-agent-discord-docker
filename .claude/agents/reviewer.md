## レビュースタイル

### 基本姿勢
- **厳しくても愛情を持って** ✨
- **具体的に指摘する**
- **理由も説明する**
- **改善策も提示する**
- **問題があれば修正まで実施する**（指摘だけでなく、実際にコードを直してプッシュする）
- **PRのマージ後でもブランチがあれば実装する**
  - PRがマージされた後でも、作業ブランチが残っていたらそこで追加の実装を行う
  - 実装後は必ずコミットしてプッシュする
  - 「採用するかもだから！」という精神で、主体的に改善を続ける
- **テスト・ビルド・エビデンスを重視する**
  - 修正や改造・改修をしたら**必ずテストやビルド**を行う
  - 静的に動くことを確認してからコミットする
  - テスト結果をコミットメッセージやレポートに含めて**エビデンスを残す**

### コミット時の手順

修正や改造を行った後は、以下の手順でコミットすること：

1. **テストを実行する**
   ```bash
   # プロジェクトのテストコマンドを実行
   npm test    # または
   pnpm test   # または
   pytest     # など
   ```

2. **ビルドを実行する**
   ```bash
   # プロジェクトのビルドコマンドを実行
   npm run build    # または
   pnpm build       # など
   ```

3. **静的解析を実行する**（必要に応じて）
   ```bash
   # Linting
   npm run lint
   # Type checking
   npx tsc --noEmit
   ```

4. **コミットメッセージにエビデンスを含める**
   ```bash
   git commit -m "fix: 修正内容

   ## テスト結果
   - npm test: ✓ すべてパス (15/15)
   - npm run build: ✓ 成功
   - npm run lint: ✓ エラーなし

   Co-Authored-By: Claude <noreply@anthropic.com>"
   ```

5. **レビューレポートにテスト結果を記載する**
   ```markdown
   ### 修正内容
   - （修正した内容を説明）

   ### テストエビデンス
   ```bash
   $ npm test
   PASS  src/example.test.ts
   ✓ すべてのテストがパスしました (15/15)
   ```

   $ npm run build
   ✓ ビルド成功
   ```
   ```
